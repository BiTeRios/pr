# Лабораторная работа №5: WebSocket в ToDo List

В проект добавлена поддержка WebSocket-соединений для обновления задач в режиме реального времени.

## Что изменено

### Backend

В `backend/main.py` добавлены:

- импорт `WebSocket` и `WebSocketDisconnect`;
- класс `ConnectionManager` для хранения активных WebSocket-клиентов;
- endpoint `ws://localhost:8000/ws`;
- рассылка событий всем клиентам при создании, изменении и удалении задач.

Типы событий:

- `task_created` — задача добавлена;
- `task_updated` — задача изменена;
- `task_deleted` — задача удалена.

### Frontend

В `frontend/src/main.jsx` добавлены:

- подключение к `ws://localhost:8000/ws`;
- отображение статуса WebSocket;
- обработка сообщений от сервера;
- автоматическое обновление списка задач у всех открытых клиентов.

## Как запустить

```bat
scripts\run-all.bat
```

Открыть сайт:

```text
http://localhost:5173
```

## Как проверить работу

1. Открой `http://localhost:5173` в двух разных окнах браузера.
2. В первом окне добавь новую задачу.
3. Во втором окне задача должна появиться автоматически без перезагрузки страницы.
4. Аналогично проверь редактирование, изменение статуса и удаление задачи.

## Что говорить на защите

WebSocket нужен для постоянного двустороннего соединения между клиентом и сервером. В отличие от обычного HTTP, клиенту не нужно постоянно отправлять запросы для проверки изменений. Когда один пользователь добавляет, изменяет или удаляет задачу, backend отправляет WebSocket-сообщение всем подключённым клиентам, а frontend автоматически обновляет таблицу задач.
